## Ships & Gear

- **Erkul Games (DPS Calculator)** — [erkul.games](https://www.erkul.games/) — Ship loadout planner and DPS calculator. The reference tool for builds. 🌐
- **FleetYards** — [fleetyards.net](https://fleetyards.net/) — Ship database with detailed specs and component slots. 🌐
- **Ship Matrix (Official)** — [robertsspaceindustries.com/ship-matrix](https://robertsspaceindustries.com/ship-matrix) — Official CIG ship specs. Authoritative but updated slowly. 🌐
- **CCU Game** — [ccugame.app](https://ccugame.app/) — CCU upgrade path finder. Calculates the cheapest chain of upgrades to reach a target ship. 🌐 ⚠️
- **Hangar.link** — [hangar.link](https://hangar.link/) — Track and manage your ship collection by linking your RSI account. 🌐 🔒 ⚠️
- **StarShip 42** — [starship42.com](https://www.starship42.com/) — 3D ship viewer with size comparisons and detailed breakdowns. 🌐 ⚠️
- **SPViewer** — [spviewer.eu](https://www.spviewer.eu/) — Ship performance database: fuel, endurance, capacitors and component stats with side-by-side comparisons. 🌐 ⚠️

## Trade & Logistics

- **UEX Corp** — [uexcorp.space](https://uexcorp.space/) — Commodities, prices, trade routes, refineries. Community-driven, very complete. 🌐 ⚠️
- **SC Trade Tools** — [sc-trade.tools](https://sc-trade.tools/) — Trade route optimizer with profit calculations. 🌐
- **Regolith Co** — [regolith.rocks](https://regolith.rocks/) — Mining co-op tracker, refinery planner and mineral price database. 🌐 🔒 ⚠️

## Databases & Reference

- **SC Unpacked** — [scunpacked.com](https://scunpacked.com/) — Raw game data extracted from each patch. 🌐 ⚠️
- **SCMDB** — [scmdb.net](https://scmdb.net/) — Star Citizen Mission Database: missions, rewards, reputation paths, blueprints and crafting resources. 🌐 ⚠️
- **CStone: Universal Item Finder** — [finder.cstone.space](https://finder.cstone.space/) — Find where any in-game item or component is sold, and at what price. 🌐 ⚠️

## Wikis & Lore

## Contributing

Contributions welcome! To add or update a tool:

1. Fork the repository.
2. Add your entry **in both** [`README.md`](README.md) and [`data.json`](data.json) following the format below.
3. If you can, also update the translated READMEs ([`README.fr.md`](README.fr.md), [`README.es.md`](README.es.md)). If you can't, that's fine — a maintainer will translate.
4. Open a Pull Request with a short justification (why is this tool useful, is it still maintained).

### Entry format — README

```
- **Tool Name** — [domain.tld](https://full-url/) — One-sentence description. <icons>
```

### Entry format — data.json

```json
{
  "id": "unique-kebab-case-id",
  "name": "Tool Name",
  "url": "https://full-url/",
  "category": "ships-gear",
  "description": "One-sentence description.",
  "flags": ["website"]
}
```

### Rules

- One tool = one entry. No duplicates across categories — pick the best fit.
- Dead links and abandoned tools will be removed during periodic reviews.
- Keep descriptions to **one sentence**, neutral tone.
